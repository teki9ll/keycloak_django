Permission System Guide
How Permissions Work

Permissions in this system are predefined and grant specific capabilities to users. When assigned to roles, they control what users can access and do within the application.

Available Permissions
Task Management
view_tasks

View tasks and task lists

manage_tasks

Create, update, delete tasks and manage assignments

System Integrations
view_integrations

View system integrations and their status

manage_integrations

Configure system integrations and external services

Administrative Access
view_admin

View administrative functions and system settings

manage_admin

Access and modify administrative functions and system settings

Quick Tips
Permissions in JWT: All assigned permissions appear in the user's JWT token
Granular Control: Assign specific permissions to different roles
Composable: Roles can have multiple permissions
Best Practice: Create descriptive permission names
