Permission System Details
How Permissions Work

Permissions in this system are predefined and grant specific capabilities to users. Each permission represents a specific action that a user can perform within the application. When users are created, they can be assigned permissions directly, and these permissions are included in their JWT tokens for authorization checks.

Available Permissions & What They Do
view_tasks
View Tasks

Allows users to view tasks, task lists, and task details. Users can browse through existing tasks and see all task information.

Capabilities:
  • Browse and read task information
  • View task details and descriptions
  • See task assignments and status
manage_tasks
Manage Tasks

Full control over task management. Users can create, edit, delete, and assign tasks to other users.

Capabilities:
  • Create new tasks
  • Edit existing tasks
  • Delete tasks
  • Assign tasks to users
  • Update task status and priority
view_integrations
View Integrations

Allows users to view system integrations, connection status, and integration settings.

Capabilities:
  • View configured integrations
  • Check connection status
  • See integration logs and errors
manage_integrations
Manage Integrations

Full control over system integrations. Users can configure, test, and manage external system connections.

Capabilities:
  • Configure new integrations
  • Edit integration settings
  • Test connections
  • Manage API keys and credentials
view_admin
View Admin Functions

Allows users to access administrative functions and view system settings and user management.

Capabilities:
  • View user management dashboard
  • See system statistics
  • Access admin reports
  • View permission assignments
manage_admin
Manage Admin Functions

Full administrative access. Users can manage other users, assign permissions, and control system settings.

Capabilities:
  • Create and manage users
  • Assign permissions to users
  • Manage user sessions
  • Access system configuration
Implementation Note

Permissions are implemented as Keycloak roles with the "permission_" prefix. This allows for easy integration with Keycloak's role-based access control system while providing a user-friendly permission interface.

JWT Token Inclusion: All assigned permissions are automatically included in the user's JWT token under realm_access.roles, making authorization checks simple and efficient.
