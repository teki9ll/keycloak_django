Permission Guidelines
Important Guidelines
  • Use snake_case: Permission names should use lowercase letters and underscores (e.g., "manage_tasks", "view_reports")
  • Be specific: Names should clearly indicate what the permission allows (e.g., "create_user" instead of "user_management")
  • Use action-oriented names: Start with verbs when appropriate (e.g., "manage_", "view_", "create_", "delete_")
  • Keep it concise: Avoid overly long permission names that are hard to remember
  • Be consistent: Follow a naming convention across all permissions
